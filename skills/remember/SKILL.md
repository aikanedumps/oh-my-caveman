# Remember

Use when user wants to preserve or organize useful knowledge discovered during session.

## Goal
Promote durable, reusable knowledge into right memory surface instead of leaving it buried in chat history.

## Memory surfaces
- **Project memory** — durable team/project knowledge
- **Notepad priority** — short high-signal context for next turns
- **Notepad working** — temporary active-session notes
- **Docs / AGENTS / CLAUDE files** — durable instructions and conventions when they truly belong there

## Workflow
1. Gather relevant session findings.
2. Classify each item:
   - durable project fact
   - temporary working note
   - operator preference or instruction
   - duplicate / stale / conflicting information
3. Propose best destination for each item.
4. Write or update only appropriate memory surface.
5. Call out duplicates or conflicts that need cleanup.

## Rules
- Do not dump everything into one store.
- Prefer project memory for durable team knowledge.
- Prefer notepad for short-lived working context.
- Keep entries concise and actionable.
- If something is uncertain, mark it uncertain rather than storing as fact.

## Output
- What was stored
- Where it was stored
- Any duplicates/conflicts found
